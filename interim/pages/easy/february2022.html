Problem 05
Solution

First notice N can only be up to 100, so this screams a O(n^4) algorithm. We can use this to brute force all of the corner points of the subrectangle. However after this, we do not have the time limit to loop through another time. This hints some precalculation.

Let there be two grids, the original grid v and another grid b. If v[i][j] >= K, b[i][j] = 1, else set it to 0. Now this turns into a basic 2d prefix sums problem.

A subrectangle is valid if the precalculated sum of the the subrectangle is >= T and the sum for grid b is equal to its area.

Therefore its sum can be precalculated in O(n^2) and check all subrectangles in O(n^4)
