Problem 30
Solution

Solve this with a greedy solution. The key observation is that there's always an optimal solution where none of the flipped ranges overlap. To show this, let's assume that two of the ranges do overlap. There are multiple ways this can happen, but the result will have a flipped range, followed by an unaffected range (that was flipped twice), followed by another flipped range. So we can replace the intersecting ranges with the two flipped ranges on either end, producing the same effect and removing this intersection.

So count the number of contiguous ranges of cows that need to be flipped. That number is the answer.

This isn't the only greedy solution, so if you have something else that is fine too.

Problem 28
Solution

The hard part of this is noticing that in a working permutation all the blue elements can the be the first numbers and the red elements the last numbers. After this just check to see if we can construct that permutation. To do that there must exist enough "large" blue number and "small" red numbers. By "large" blue number, I mean the ith greatest blue number must be greater than or equal to the number i. By "small" red number I mean the ith greatest red number must be less than or equal to the ith greatest number from 1 to n.

Problem 26
Solution

There are three cases to consider.

N = 1 and M = 1. This is really easy, x = 1 and y = 1, done, k = 0.

N = 1 or M = 1. This is a bit harder but still not too bad. x or y has to be 1 because one dimension of the grid has length 1. The second dimension can be found by guessing (1, 1). The difference in one dimension is zero, so the distance between the guess and the hidden point is the y coordinate. (well technically the y coordinate minus 1). Therefore k = 1.

N > 1 and M > 1. Neither of the above techniques work. Let g = min(N, M). Place points (1, 1) and (1, g) to get responses from the computer, b1 and b2. Using the formula for manhattan distance, |x - 1| + |y - 1| = b1 and |x - 1| + |y - g| = b2. This is a linear system of equations with two unknowns and therefore can be solved to find the values of x and y. Therefore k = 2.

Really this entire problem could be thought of as applying the above formula for manhattan distance instead of the two strategies I gave (a good way to start trying to solve it). For the fist two cases: In the first case, both x and y are known so there are zero needed equations. If the second case, one of x or y is known so there need be one one equation, |x - 1| = b1 or |y - 1| = b1, a linear equation, something easy to solve.
