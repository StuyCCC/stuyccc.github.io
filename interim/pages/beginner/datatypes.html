DATA TYPES

Printing VS Returning

Previously, we showed that a C++ program can ‘print’ a value, such as “Hello World!”, to the terminal. What it’s doing is showing the programmer what a specific value is-—for instance, if a variable changes, it’s helpful to see how it’s changing as the program runs. This is different from returning, which is the value taken on by a function y = f(x). Note that functions can only return a single value each time it’s run, but we can print values as many times as we want.

The values returned by functions can be used again elsewhere in the program. On the other hand, printed values cannot be used or stored. They are only shown to the user.

When submitting code to an automated grader, answers must be printed. There is no other way to show the grader an answer; if it’s returned internally, an outside source would have no way of knowing.

Syntax For Printing

The syntax for printing is ‘cout << x;’, where x can be any value from a number to a word. Because cout doesn’t care about what x is, it’s type blind.

We can also print multiple things in a chain with cout << x << y. Note that it will print exactly what we tell it to; so this would print xy with no space between them. If we wanted to put a space in between x and y and add a newline at the end, we would have to type ‘cout << x << “ “ << y << “\n”;’.

Variables

In math, we might declare a variable like ‘let x = 5’. However, the computer doesn’t know what x is unless we declare its type. We can do it as such:
int x = 5;

This allocates a box named ‘x’ which the computer now knows will store an integer. Once we declare its type once, we don’t have to specify it again, and we can just refer to it as ‘x’. Of course, the type and name can be changed into anything the user wants. In general, the syntax is
type name = value;
where the value can be a number, string, boolean, or another variable.

A note on the equals sign: in math, an equals sign indicates that the left and right sides of the equation are equal in value. However, in C++ it’s more of an assignment operator, meaning we are putting the value on the right side into the variable on the left. We can think of like this:
int x ← 5;

We are putting the value 5 inside of x. As such, variables can be reassigned multiple times, with different values being put in each time.

The actual equivalency operator in C++ that is like in math is the double equal sign: '=='

Type Commutativity

After a bit of experimentation, we can show that the types {int, double, char, bool} are all commutable, meaning that they can be converted into each other. There are some things to look out for:

  • When putting a double into an int, the values after the decimal point are truncated (basically removed).
  • Putting values into bools turns it into either a 0 or a 1.
  • The integer value of chars are its values in the ASCII table.

Standard arithmetic operations work on these values, which we will explore more later, but for now you can experiment on your own.
