Problem 29
Solution

To solve, lets iterate over possible values which we can make half the array. The possible values to make the array are just every value in the array. So, just consider some value m in the array. Count the amount of numbers equal to m and let that equal c, numbers which we don't have to change. Then find the difference between m and all remaining possible a_i, (the a_i > m). For each a_i compute all the possible divisors of a_i - m. These are the possible values for k. As we iterate over the a_i's, keep track of the amount of a_i - m each k divides. Now, for that m, we know that all k such N/2 - c <= # a_i such that that k divides a_i - m will be valid values of k. We can now take the maximum valid k over all values of m as our answer.

Problem 27
Solution

There is more than one solution but I did it with DP. let dp[i][j][k] = # of ways to make the least k significant digits of n

The index j is 0 if there is a carry which affects this current index. The index k is 0 if there is a carry which will affect the next index.

Initialize dp to zeros, and notice dp[1][1][0] = 1. We can now write out the transitions. Let d = the kth least significant digit of n.

We can then write, to construct the dp array.

dp[1][1][k+1] = (d+1) * dp[1][1][k] + d * dp[0][1][k]

dp[0][1][k+1] = (d+1) * dp[1][0][k] + d * dp[0][0][k]

dp[1][0][k+1] = (9 - d) * dp[1][1][k] + (9 - d + 1) * dp[0][1][k]

dp[0][0][k+1] = (9 - d) * dp[1][0][k] + (9 - d + 1) * dp[0][0][k]

The top two statements represent adding together two digits at in the two numbers we are summing such that their sum is less than 10. The second two statements represent adding two digits whose sum is greater than 9 (the addition creates a carry).

I'll work through one line and hopefully the others will be similar enough to figure out:

dp[1][1][k+1] = (d+1) * dp[1][1][k] + d * dp[0][1][k]
^^^^^^^^^^^^^
This represents the number of ways to get to the kth least significant digit such that there are no carries which will affect that kth digit or the k+1th digit after it.

dp[1][1][k+1] = (d+1) * dp[1][1][k] + d * dp[0][1][k]
                        ^^^^^
number of ways to get two numbers which sum to d

dp[1][1][k+1] = (d+1) * dp[1][1][k] + d * dp[0][1][k]
                                    ^^^^^^^^^
if numbers sum to d, then there was no carry affecting this digit, and also we want not a carry to affect the next digit, so we multiply by the appropriate array value.

dp[1][1][k+1] = (d+1) * dp[1][1][k] + d * dp[0][1][k]
                                                       ^^^^^^^^^^^^^
very similar to last term, but now numbers are adding to d-1 because we assume a carry happened which is affecting this digit
