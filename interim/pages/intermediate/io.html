INPUT AND OUTPUT

Python

IO in python comes down to using the input() function and using python's string parsing.

input() returns a string containing one line of input from standard in. Standard in is one way a program communicates with the enviorment around it. When you run a program, input() will cause the program to wait until a line of text is found in standard in.

If you are just running a python program in terminal with the command python program.py, it will wait for you to type some characters and then press enter (or return, or whatever will input a new line). At that time, input() will return the string of characters you typed. 


s = input()
# run the program and type "hello"
# then press enter/return/whatever the equivalent for you is
print(s)
# hello is printed as s is the string "hello"

Now that you have a string, the most common things to do are to use strip() to get rid of trailing whitespace and split() if you want to turn a string of space separated elements of a list into a list. Use list/generator comprehension if you can.

I'll follow with writing some examples of python IO: 


# the input is in the format
# N
# where N is an integer
N = int(input())

# input() will read from standard in the string representation
# of N
# int() will parse that string for the integer N, and return
# that integer
            

# the input is in the format
# N M
# where N and M are both integers
N, M = [int(x) for x in input().split()]

# this uses input() to get the two integers in their string
# representation
# then split turns those integers into a list ['N', 'M']
# using list comprehension this turns the list [N, M]
# notice N and M are integers now
# finally these get set to the correct variables, N and M
            

# the input is a list of integers of format
# a1 a2 a3 ... an
# ai is an integer
A = [int(x) for x in input().split()]

# this is similar to the last example
# input() will get the string of space separated integers
# split() will turn that string into an array of strings
# representing integers
# using list comprehension and int
# we turn this array of strings into an array of integers

Final Note

Just using input() can be slow, so instead we use sys.stdin.readline. This doesn't do exactly the same thing as input() (it is one of those things to use for competitive programming but not in general) but for the use of competitive programming it is fine. This amounts to adding the following to the your program: 


import sys
input = sys.stdin.readline

Java

Java has a very wide variety of methods to do IO, which have tradeoffs between being complicated and being slow.

Recommended imports

We will need a few imports to allow some of the following methods to work. Some of the methods require only either util or io, and we do not need everything in both libraries, but it's helpful to have access to other stuff in here. This comes with the disclaimer that doing full imports of these libraries are bad code practice in actually making projects. 


import java.util.*;
import java.io.*;

These two imports should go at the top of your file.

Scanner

Scanner has an advantage in not requiring any extra methods to directly read into important primitive types like integers, doubles, longs, etc. It's disadvantaged in speed, though.

To create a Scanner object: 


Scanner s = new Scanner(System.in);

To directly read certain types: 


String str = s.next(); // Next word from standard in (whitespace separated)
int a = s.nextInt(); // Next integer number
double d = s.nextDouble(); // Next decimal number
String line = s.nextLine(); // Next full line

If the problem doesn't give you a number that lets you see how far you need to read, you can check the boolean s.hasNext(), and if you need to check if the next word satisfies a type, you can check the function hasNext[Type](), i.e., s.hasNextInt()

BufferedReader

BufferedReader is significantly faster than Scanner, but comes at the cost of only being able to return strings of lines or single characters. When you combine it with string processing methods we'll learn later, though, you'll be able to get the best of both worlds.

To create a BufferedReader: 


BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

To read lines: 


String line = br.readLine();

If there are no more lines to read, br.readLine() will return null, which is helpful for counting lines.

A fairly important thing to take note of is that every function that contains another function that throws IOException must itself either throw or catch IOException.

For us, since we're not concerned with following proper code practices, we can stick to just adding throws IOException anywhere that it's needed. For example: 


public class Foo {
    public static void main(String[] args) throws IOException {
        String myPassword = "p4ssw0rd";
        System.out.println(correctPassword(myPassword));
        //                 ^ This throws IOException, so we throw IOException in main
    }

    public static boolean correctPassword(String match) throws IOException{
        // This throws IOException, so we throw IOException in correctPassword()
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        String inputPassword = br.readLine();

        return match.equals(inputPassword);
    }
}

Most of the time, you will only need to use BufferedReader in your main function, so you are usually safe with just sticking throws IOException next to main.

Processing Strings of Lines

Now that we have our lines, we have to get the right data from them.

Converting Strings to Other Types

If your String is in the correct format (i.e., for an integer, all digits), then you can use each type's parse method, which will return the corresponding value in the correct type. 


String readInLine = "52383";
int n = Integer.parseInt(readInLine); // 52383
String secondReadInLine = "5.2383";
double d = Double.parseDouble(secondReadInLine); // 5.2383
Splitting Lines with Spaces Into Words

Option 1: Using String.split(" ") will return an array of Strings, separated by the given string argument - for our use case, spaces. 


String s = "Hello, I am Sam Belliveau";
String[] words = s.split(" "); // ["Hello,", "I", "am", "Sam", "Belliveau"]

Option 2: Using StringTokenizer can grant you slightly more speed and behavior like Scanner, except only on a single String.

To create a StringTokenizer: 


StringTokenizer st = new StringTokenizer("Hello, I am Sam Belliveau");

To get the next word in the String, use st.nextToken(): 


String first = st.nextToken(); // Hello,
String second = st.nextToken(); // I
String third = st.nextToken(); // am
String fourth = st.nextToken(); // Sam
String fifth = st.nextToken(); // Belliveau
boolean hasNone = st.hasMoreTokens(); // false

An example

Say we wanted to read n line-separated pairs of space separated integers and print out each pair's sum. On the first line, we are given n. 


4
5 6
1 2
-5 323
862983 1239019


import java.util.*;
import java.io.*;

public class Demo {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st;
        int n = Integer.parseInt(br.readLine());
        for (int i = 0; i < n; i++) {
            st = new StringTokenizer(br.readLine());
            int a = Integer.parseInt(st.nextToken());
            int b = Integer.parseInt(st.nextToken());
            System.out.println(a + b);
        }
    }
}
